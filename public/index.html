Real Bugs We Found & Fixed

🐛 BUG #1: NULL POINTER CRASH

addToCart() with Unknown Product ID Crashes App

Problem: Calling addToCart(999) with a non-existent product ID crashed the application with an undefined lookup error.

Impact: User submits invalid data → app breaks completely. Production outage risk.

How We Found It: Loop test called addToCart() with invalid IDs (boundary value analysis). Caught immediately.

Fix: Added guard clause to validate product ID exists before processing.

Cost Saved: Would have cost $5k-20k to debug in production.

🐛 BUG #2: NAN INJECTION

changeQty() Injects NaN Into Cart Total

Problem: Calling changeQty(999, +1) with unknown product ID left NaN in the cart total.

Impact: Cart displays "$NaNaN.NaN" → checkout impossible → lost sales.

How We Found It: Loop test LOOP-05 added the same product 25 times and observed total = $NaN.

Fix: Added existence check in changeQty() before math operations.

Cost Saved: Would have been a customer-facing issue costing reputation + refunds.

🐛 BUG #3: RACE CONDITION

Rapid add/remove Operations Corrupt Cart State

Problem: Calling addToCart() and removeItem() in rapid succession left items in cart that should have been deleted.

Impact: Cart state becomes inconsistent with user intent. Compliance issues (charged for items they didn't want).

How We Found It: Loop tests LOOP-03 and LOOP-06 ran add-then-remove 15-20 times and detected leftover items.

Fix: Atomic state updates with proper locking mechanism.

Cost Saved: Would have caused customer support tickets, refund requests, trust loss.
