📋 Project Overview

QA_AGents is a production-grade, multi-agent QA automation framework. It demonstrates a complete pipeline that:

  • Takes GitHub issues as input
  • Generates complete Playwright test suites automatically
  • Executes tests across multiple browsers (Chrome, Firefox, Safari)
  • Self-heals broken selectors when the UI changes
  • Reports results with gate decisions (PASS/FAIL)

🏗️ Core Components

7-Agent Pipeline

Each component is a specialized agent in a linear chain:

  1. Ingestor: Parses GitHub issues, extracts requirements
  2. Planner: Designs test strategy (BVA, EP, pairwise, etc)
  3. Generator: LLM writes runnable Playwright + TypeScript code
  4. Reviewer: Critic agent audits quality, suggests improvements
  5. Runner: Executes Playwright CLI, captures results
  6. Healer: Detects selector drift, repairs broken locators
  7. Reporter: Rule-based gate (PASS/FAIL), narrative summary
Key Rule: Never let the LLM make gate decisions. ReporterAgent._gate() is pure code-based logic: any P0 failure or pass rate below 90% → FAIL.

✨ Best Practices

🎯 Auto-Waiting

No hardcoded waits. Use expect(locator).toHaveText(), toBeVisible(), toHaveCount() for dynamic, intelligent retry logic.

🔍 Accessibility-First Locators

Prioritize data-testid, aria-live, and getByRole over fragile CSS classes. Resistant to minor DOM refactors.

🔐 Test Isolation

Each test gets a fresh browser context via test.beforeEach(). No state bleed between tests.

⚡ Performance

Eliminated 19 hardcoded waits. Suite time dropped from 11.6m to 5.4m (2x faster).

🚀 Quick Start

Install

pip install -r requirements.txt npm i -D @playwright/test allure-playwright npx playwright install

Run Tests

npm run test:store # Full suite (3 browsers) npm run test:store:smoke # @smoke only (fast) npm run test:store:baselines # Capture visual baselines

Scaffold New Project

python3 scripts/scaffold-suite.py \ --url https://myapp.com \ --prefix myapp \ --auth npm run test:myapp

View Reports

open playwright-report-store/index.html # Playwright HTML report open store-qa-report.html # Custom QA report with history
